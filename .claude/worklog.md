## 2026-07-22 — Converted parnell_repro.qmd into a working Shiny app

**Done:** Rebuilt `ParnellRepro/app.R` from scratch. The old file was a scaffold that
referenced variables that never existed (`df_herd`, `lactation_group`,
`pregnancy_status`, `days_in_milk`) and could not run. The new app ports every KPI
calculation from `parnell_repro.qmd` verbatim — insemination rate, conception rate,
pregnancy rate (all by lactation group over rolling 21-day periods), monthly rebreeds,
the abortion cohort, and % pregnant by 100/150/200 DIM — laid out as bslib tabs with a
sidebar. Verified with `shiny::testServer` (all 16 outputs render, reactives respond to
input changes) and a live browser render on port 7654.

**Why:**
- The KPI logic is copied line-for-line from the qmd chunks so the app's numbers match
  the report. Interactivity was added only where it's meaningful and safe: voluntary
  waiting period, number of periods, which lactation groups to plot, and how many recent
  periods to exclude from trend graphs (default 2, reproducing the report's rule).
- Dropped the qmd's `fxn_load_os_fxns()` / `fxn_denos_sarah()` / `fxn_denominators_monthly()`
  setup: those source functions from GitHub and compute denominators that *no visible
  output actually uses*. Omitting them lets the app run offline and removes the GitHub
  API-rate-limit failure mode.
- Dropped the qmd's "R Mystery" chunk — it was exploratory `head()` output and printed
  animal-level rows, so it doesn't belong in a dashboard.
- Tables always show all three lactation groups (avoids brittle dynamic gt columns); the
  lactation-group and drop-recent controls affect only the graphs.

**Gotchas:**
- The app relies on `data/intermediate_files/{events_formatted,animals,animal_lactations}.parquet`.
  Those were lost when the earlier history purge (`git filter-branch`) force-reset the
  working tree, then regenerated by running the pipeline. They're gitignored now, so a
  fresh clone must run `step0_master_processing_my_data.R` before the app will start (the
  app fails fast with a clear message if they're missing).
- `here::here()` inside `ParnellRepro/` still resolves to the project root because of the
  root `.Rproj`; don't "fix" the paths to be relative to the app folder.
- Expect ~21k `max(): no non-missing arguments` warnings at startup — cows with no
  BRED/OPEN/DNB event; the code converts the resulting `-Inf` to `NA` immediately. Same
  behavior as the report; harmless.

**Next:** The natural follow-on is the anonymized cross-herd comparison app that this
folder was originally meant for (compare CR/PR/abortion across herds) — still unstarted.
Also still open from before: the pregnancy-rate numbers have not been validated against
DC305's own repro summary.
